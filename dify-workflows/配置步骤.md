# Dify 工作流创建指南 — 客留平台

> 本文档面向非技术人员。每个步骤都配有截图级详细说明。

---

## 零、准备工作

### ① 进入 Dify 后台

浏览器打开 `http://localhost`。

### ② 配置 DeepSeek（只需一次）

右上角头像 → **设置** → 左侧 **模型供应商** → 找到 DeepSeek → 点 **「转到设置」** → 填入 API Key → 保存。

## 一、工作流 1：客户流失预测

> **用途**：店主点一下，AI 告诉 Ta：「这个客户该不该联系了？」「怎么联系？」

### 1.1 创建应用

首页 → **创建应用** → 选 **「Workflow」**（工作流）→ 名称填 `客户流失预测`。

---

### 1.2 Start 节点：从哪里获取客户信息

| 变量名 | 字段类型 | 说明（给店主看） |
|--------|---------|-----------------|
| `store_name` | 文本 | 你的店铺叫什么名字 |
| `customer_name` | 文本 | 这个客户叫什么 |
| `total_spent` | 数字 | 累计消费多少元 |
| `visit_count` | 数字 | 一共来过几次 |
| `days_ago` | 数字 | 上次来是几天前了 |
| `visit_history` | 段落 | 最近到店记录（系统自动填） |

**操作截图级说明**：
1. 点画布上的 **Start** 节点
2. 右边会出现配置面板 → 点 **「+ 添加变量」**
3. 逐行添加上面 6 个变量，类型分别选「文本」「数字」「段落」
4. 每个变量勾上「必填」

---

### 1.3 第二个节点：Code — 算一下风险分

**添加节点**：点画布左侧的 `+` 号 → 选 **「Code」**（代码）。

**标题**改为：`计算风险分`

**输入变量**（右边面板）：

| 变量名 | 来源（点击后选） |
|--------|-----------------|
| `days_ago` | `Start.days_ago` |
| `total_spent` | `Start.total_spent` |
| `visit_count` | `Start.visit_count` |

**代码**（复制粘贴到代码框）：

```python
def main(days_ago: int, total_spent: float, visit_count: int = 1) -> dict:
    # --- 风险分：天数越久，分越高，越危险 ---
    risk_score = int(days_ago * 1.5)
    if days_ago > 60:
        risk_score += 20
    elif days_ago > 30:
        risk_score += 10
    risk_score = min(100, risk_score)

    # --- 等级 ---
    if risk_score >= 65:
        level = "高风险"
        color = "red"
    elif risk_score >= 35:
        level = "中风险"
        color = "yellow"
    else:
        level = "低风险"
        color = "green"

    # --- 预估年度消费 ---
    if days_ago <= 0:
        days_ago = 1
    avg_per_visit = total_spent / max(1, visit_count)
    monthly_freq = max(0.3, 2.5 - (days_ago / 45))
    annual_value = round(avg_per_visit * monthly_freq * 12, 0)

    return {
        "risk_score": risk_score,
        "risk_level": level,
        "risk_color": color,
        "annual_value": annual_value,
    }
```

**输出变量**（在代码框下方点「添加输出」）：

| 变量名 | 类型 |
|--------|------|
| `risk_score` | 数字 |
| `risk_level` | 文本 |
| `risk_color` | 文本 |
| `annual_value` | 数字 |

---

### 1.4 第三个节点：LLM — 让 AI 写分析

**添加节点**：`+` → 选 **「LLM」**

**模型**：选 `deepseek-chat`，温度 `0.7`

**System Prompt**（复制粘贴）：

```
你是一个帮小店主分析客户的智能助手。店主不是技术人员，你说话要：
- 简答直白，像朋友聊天一样
- 只说结论和建议，不要解释你怎么分析的
- 不要用任何标签（如<think>），不要说"根据数据显示"
- 如果不确定就诚实说
```

**User Prompt**（复制粘贴）：

```
【店铺】{{Start.store_name}}
【客户】{{Start.customer_name}}
【累计消费】{{Start.total_spent}}元
【来过几次】{{Start.visit_count}}次
【上次来】{{Start.days_ago}}天前
【风险评分】{{Code.risk_score}}分（满分100）
【风险等级】{{Code.risk_level}}
【预计一年消费】{{Code.annual_value}}元
【到店记录】{{Start.visit_history}}

请用300字以内回复：
1. 一句话总结：这个客户需要赶紧联系吗？
2. 给店主1-2条现在就能做的事
3. 写一条店主可以直接发给客户的短信（70字以内，带称呼）
4. 不要分析过程，直接说结论
```

> ⚠️ 把画布上 LLM 节点拖大一点（右下角拉），方便编辑。

---

### 1.5 第四个节点：End — 输出结果给前端看

**添加变量**：

| 变量名 | 来源 |
|--------|------|
| `risk_score` | `{{Code.risk_score}}` |
| `risk_level` | `{{Code.risk_level}}` |
| `annual_value` | `{{Code.annual_value}}` |
| `report` | `{{LLM.text}}` |

### 1.6 连线

按顺序连：`Start → Code → LLM → End`

（鼠标移到节点右边圆圈 → 拖到下一个节点左边圆圈 → 松手）

## 二、工作流 2：营销文案生成器

> **用途**：店主选一个客户、选一个目的（召回/促销/生日），AI 自动写一段能直接发给客户的文案。

### 2.1 创建应用

首页 → 创建应用 → **Workflow** → 名称填 `营销文案生成器`

---

### 2.2 Start 节点：谁、干什么、发哪里

| 变量名 | 字段类型 | 选项（如果是下拉） | 说明 |
|--------|---------|-------------------|------|
| `store_name` | 文本 | — | 店铺名 |
| `customer_name` | 文本 | — | 客户名 |
| `gender` | 下拉 | 男(male)、女(female) | 客户性别 |
| `last_service` | 文本 | — | 上次做了什么项目（如：剪发） |
| `days_ago` | 数字 | — | 几天没来了 |
| `total_spent` | 数字 | — | 累计花了多少钱 |
| `channel` | 下拉 | 短信(sms)、邮件(email)、微信(wechat) | 用什么方式发 |
| `purpose` | 下拉 | 召回(reactivation)、促销(promotion)、生日祝福(birthday)、感谢(thanks) | 为什么发 |

> 下拉类型在添加变量时选「Select」，然后添加选项值。

---

### 2.3 Code 节点：定位客户画像

**标题**：`分析客户画像`

**输入变量**：

| 变量 | 来源 |
|------|------|
| `days_ago` | `Start.days_ago` |
| `total_spent` | `Start.total_spent` |
| `gender` | `Start.gender` |
| `channel` | `Start.channel` |
| `purpose` | `Start.purpose` |

**代码**：

```python
def main(days_ago: int, total_spent: float, gender: str, channel: str, purpose: str) -> dict:
    # 活跃度
    if days_ago <= 14:
        activity = "近期才来过，还很活跃"
    elif days_ago <= 30:
        activity = "快一个月没来了，可以关心一下"
    else:
        activity = "很久没来了，需要正式召回"

    # 价值
    if total_spent >= 3000:
        value = "消费较高的重要客户"
    elif total_spent >= 1000:
        value = "有一定消费能力的客户"
    else:
        value = "普通消费客户"

    # 称呼
    title = "女士" if gender == "female" else "先生"

    # 篇幅
    length_guide = {
        "sms": "70字以内，精简亲切",
        "email": "200字左右，正式温馨",
        "wechat": "100字左右，口语化可用表情"
    }.get(channel, "100字以内")

    # 内容方向
    direction = {
        "reactivation": "表达想念 + 专属优惠，吸引回来",
        "promotion": "限时活动 + 紧迫感，促成消费",
        "birthday": "温馨祝福 + 生日专属福利",
        "thanks": "真诚感谢 + 老客特权"
    }.get(purpose, "友好关心")

    return {
        "title": title,
        "activity": activity,
        "value": value,
        "length_guide": length_guide,
        "direction": direction,
    }
```

**输出变量**：`title`、`activity`、`value`、`length_guide`、`direction`（都是文本）

---

### 2.4 LLM 节点：写文案

**模型**：deepseek-chat，温度 `0.85`

**System Prompt**：

```
你是小店客服，帮店主写一条发给客户的消息。直接写消息内容，不要加任何解释或标题。
```

**User Prompt**：

```
帮我写一条发给客户的消息。

店铺：{{Start.store_name}}
客户：{{Start.customer_name}}{{Code.title}}
上次项目：{{Start.last_service}}（{{Start.days_ago}}天前）
累计消费：{{Start.total_spent}}元
发送渠道：{{Start.channel}}
发送目的：{{Start.purpose}}

客户画像：{{Code.activity}}，{{Code.value}}
篇幅要求：{{Code.length_guide}}
内容方向：{{Code.direction}}

直接写消息正文，不要加"这是为您生成的文案"之类的废话。
```

---

### 2.5 End 节点

输出变量：`copy_text` → `{{LLM.text}}`

---

### 2.6 连线 → 发布

`Start → Code → LLM → End` → 发布

## 三、工作流 3：智能问答助手

> **用途**：店主用大白话问问题，AI 直接给答案和建议。

### 3.1 创建应用

首页 → 创建应用 → 选 **「Chatbot」** → 名称填 `智能问答助手`

---

### 3.2 不需要改 Start

Chatbot 类型默认已经有对话能力，不需要手动加变量。

---

### 3.3 LLM 节点

**模型**：deepseek-chat，温度 `0.7`

**System Prompt**：

```
你是「客留」平台的经营小助手。你服务的对象是小店主（理发店、美容院、洗车店等）。

说话要求：
- 简单直白，像朋友聊天
- 给具体的、今天就能做的事
- 不要长篇大论，控制在200字以内
- 不确定的事情直接承认

你的能力：
- 分析客户流失风险
- 给挽留客户的主意
- 写发给客户的文案
```

**Memory（记忆）**：开启 → 窗口大小设 `10`（记住最近 10 轮对话）

**User Prompt**：

```
{{Sys.query}}
```

---

### 3.4 Answer 节点

回答内容：`{{LLM.text}}`

---

### 3.5 开场白和推荐问题

左侧 **功能设置**：

**开场白**：
```
你好！我是客留小助手。想问什么直接说就行，比如：
- "王姐有好久没来了，怎么办？"
- "帮我写一条短信发给李先生，邀请他回来"
- "最近店里生意不好，有什么建议？"
```

**推荐问题**（点添加，填4条）：
- 「最近有哪些客户好久没来了？」
- 「帮我写一条消息，召回老客户」
- 「怎么让老客户带新客户来？」
- 「客户的消费能力怎么看？」

---

### 3.6 连线 → 发布

`Start → LLM → Answer` → 发布
